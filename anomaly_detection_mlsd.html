Part 2. Frame the Problem as an ML Task

Defining the ML objective

According to the statement, we can define the ML objective as accurately classify posts as harmful or not.

Specifying the system's input and output

The input of this system is a post, and the output is the probability that the post is harmful.

Choosing the right ML category

Fusion Method

To make accurate predictions, the system should consider all modalities. Two commonly used fusing methods to combine heterogeneous data are late fusion and early fusion.

With late fusion, each modality is processed independently, and their predictions are combined to make a final prediction. [优点] This allows independent model training and improvement. [缺点] However, it requires separate training data, which can be time-consuming and expensive. In addition, it may fail when each modality is benign but harmful content arises from the combination of benign modalities.

With early fusion, the modalities are combined first, and then the model makes a prediction. [优点] Thus, it is unnecessary to collect training data separately for each modality, we only need to collect training data for the whole model. In addition, if each modality is benign, but their combination is harmful, then the model can potentially capture this. However, learning this task is more difficult.

To capture posts that may be harmful overall, even if each modality is benign on its own, we choose early fusion. Additionally, considering the size of training data, the model has enough data to learn the task.

Classifer

Besides the fusion method, we also need to consider the type of classifier. There are three main options: One binary classifier per harmful class, Multi-label classifier, and Multi-task classifier.

With One binary classifier per harmful class, each harmful class has its own binary classifier, making it easy to explain why a post was flagged and allowing independent model improvements. However, training multiple models is expensive and time-consuming.

With multi-label classifier, asingle model predicts probabilities for multiple harmful classes, reducing training and maintenance costs. However, since all classes share the same model, input features may not be transformed optimally for each class, potentially affecting accuracy.

With Multi-task classifier, a single model learns multiple harm classes. Muti-task model uses a shared model to capture the fundamental relationship between users and items, generating a shared feature representation. This shared representation is then used by seperate modules to predict metrics for each task. This approach is cost-effective as it uses a single model, efficient by avoiding redundant computations, and data-efficient since training data from one task benefits others, especially when data is limited.

Because of these advantages, we employ a multi-task classification method.

Part 4. Model Selection

With such features, we move to models development part. Since we adopt an early fusion approach, and our detection task involves multiple data modalities (text, images, videos, and structured data), we design a two-phase model. In the first phase, we use representation learning to extract embeddings from unstructured data. In the second phase, we concatenate these embeddings with structured features and feed them into a multi-task classifier to estimate the anomaly probability.

To convert text into embeddings, we can choose between statistical methods and machine learning (ML)-based models. Statistical methods are suitable for small datasets, offering efficiency and interpretability. While ML-based models can capture complex linguistic patterns and handle large-scale text data. Since our task is complex, we adopt an ML-based approach for this task. Among ML-based approach, Word2Vec and Transformer-based models are commonly used. Word2Vec assumes that a word’s meaning is determined by its context. It trains a neural network to learn the relationship between words and their surrounding context, generating word embeddings. While effective for many NLP tasks, it has several limitations: it generates static embeddings, struggles with polysemous words (words with multiple meanings); it only captures local context, unable to model long-range dependencies; it does not retain word order information, and it cannot handle out-of-vocabulary (OOV) words. In contrast, Transformer-based models generate contextual embeddings, leveraging self-attention to capture relationships between words across an entire sequence. This enables a deeper understanding of meaning. However, Transformers come with challenges, including high computational complexity, the need for large-scale training data, and difficulty in interpretability. Given that our task requires understanding complex contexts, we choose a Transformer-based model to ensure richer text representations and improved performance.

To convert images into embeddings, we can choose between Convolutional Neural Networks (CNNs) and Vision Transformers (ViTs). CNNs are highly efficient, requiring fewer parameters, making them suitable for small to mid-sized datasets and real-time applications. However, they struggle with long-range dependencies, limiting their ability to capture global patterns in complex images. On the other hand, ViTs leverage self-attention mechanisms to model global patterns effectively, leading to richer feature representations. However, they require large-scale datasets and high computational resources, making them more suitable for cloud-based and large-scale models. For our task, as we need to capture global patterns and work with large-scale data, we choose ViTs.

For the multi-task classifier, neural networks are a common choice. When selecting a neural network, it is essential to design its architecture and optimize key hyperparameters, such as the number of hidden layers, activation functions, and learning rate. These hyperparameters significantly impact the model's performance and are typically fine-tuned through hyperparameter tuning. Hyperparameter tuning is the process of finding the optimal values for hyperparameters to achieve the best model performance. A common approach is grid search, which systematically explores different combinations of hyperparameter values. This involves training a separate model for each combination, evaluating their performance, and selecting the configuration that yields the best results. To construct the training data, we use two labeling methods: manual labeling and natural labeling. In manual labeling, human annotators carefully review and label posts, ensuring high accuracy but at the expense of time and cost. On the other hand, natural labeling leverages user reports to generate labels automatically, enabling faster data collection but introducing some label noise. To balance between accuracy and efficiency, we can use naturally labeled data for training, allowing us to scale the dataset quickly, while use manually labeled data for evaluation, where label quality is a top priority. Since each task is a binary classification, we use cross-entropy as the standard loss function. The overall loss is then computed by aggregating the task-specific losses.

Part 5. Evaluation

Offline Metrics

  • Precision (correct rate among hit), Recall (true positive among all positive)

  • F1 score: 2×Precision×RecallPrecision+Recall

  • PR-AUC (ROC-AUC): summarize the trade-offs between precision and recall (recall and false positive)

Online Metrics

  • Prevalence: the extent to which we miss harmful content

    • The proportion of "harmful posts we didn’t prevent" out of "all posts on the platform"

    • = Number of harmful posts we didn’t prevent / Total number of posts on the platform

    • But it does not show how many people were affected by those posts

  • Harmful impressions

    • = Total impressions of harmful posts / Total impressions of all posts on the platform

  • Valid appeals: the extent to which we mistakenly flag content as harmful. A higher value suggests that the system may be too strict.

    • Percentage of posts that were detected harmful, but appealed and reversed.

    • = Number of reversed appeals / Number of harmful posts detected by the system

  • Proactive Rate: the extent to which we proactively detect harmful content. A higher value indicates strong detection capabilities, meaning the system can identify and remove harmful content before users report it, rather than relying on user reports.

    • Percentage of harmful posts found and deleted by the system before users report it.

    • = Number of harmful posts detected by the system / (Number of harmful posts detected by the system + reported by users )

  • User reports per harmful class.

Part 6. Serving

After defining the appropriate metrics and training a model, the next step is deploying the model to production. In general, there are three main components : detection service, violation enforcement service, and demoting service.

Given a new post, the detection service predicts the probability of harm.

If the detection service predicts harm with high confidence, the violation enforcement service immediately takes down a post. It also notifies the user why the post was removed.

If the harmful content detection service predicts harm with low confidence, the demoting service temporarily demotes the post in order to decrease the chance of it spreading among users.

Then, the post is stored in storage for manual review by humans. The review team manually reviews the post and assigns a label from one of the predefined classes of harm. We will use these labeled posts in future training iterations to improve the model.

(Data) Training Using Imbalanced Data

  • One-class learning:

    仅仅用aberrant/positive samples 来训练，让classifier仅学得aberrant/positive samples 的特征

  • Under-sampling method:

    Reduces the size of the majority class (e.g., negative samples) by removing negative samples.

    Over-sampling method:

    Increases the size of the minority class (e.g., positive samples) by duplicating positive samples

  • Weighted loss:

    在损失函数中对正负样本赋不同权重：

    Loss=α⋅Losspositive+(1−α)⋅Lossnegative α=0.8 时，增加对正样本的关注。

  • Contrastive learning:

    用 contrastive loss 来 增加同类样本内的相似性

  • Ensemble learning:

    • In boosting, the classifiers are trained iteratively. To increase the prediction accuracy, the training data in each iteration is dependent on the prediction performance of the previous classifier, the algorithm is forced to choose the misclassified samples in the previous classifier more often or weight them more heavily. Thus, the boosting algorithm can be adapted to address the imbalanced data problem by adding more weights on aberrant samples in each iteration.

    • For bagging, the algorithm first samples subsets from the original data and trains classifiers independently, then makes the final result by combining the predictions with equal weight. When it is adapted to deal with the imbalance problem, instead of random sampling, several sampling strategies such as under-sampling can be used to ensure heavier weights of aberrant samples.
