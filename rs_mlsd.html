Specifying the system's input and output

The input of this system is a user, and the output is a ranked list of items sorted by their relevance scores.

Choosing the right ML category

There are several ways to approach a recommendation problem:

  • Lightweight Embedding-Based Models (Representation-Based Models)

    These models learn embeddings for users and items, then recommend based on the similarity between those embeddings.

    • Pros: Fast and scalable: Easy to handle large candidate sets due to efficient approximate nearest neighbor searches (ANN).

    • Cons:

      • Relying solely on similarity, not fully optimize business objectives like click-through rates or engagement;

      • Not consider complex feature interactions, may less precise in recommendation.

  • Complex Feature Interaction Models (Interaction-Based Models)

    These models incorporate multiple features (such as user attributes, item metadata, context) and explicitly/implicitly learn the interactions among them.

    • Pros:

      • Higher accuracy: By leveraging multi-dimensional feature interactions, the model can capture rich, non-linear relationships.

      • Optimizes business objectives: Can directly optimize for clicks, conversions, or other downstream metrics.

    • Cons:

      • Computationally expensive.

      • Greater complexity: More hyperparameters, bigger data requirements, and potential overfitting without proper regularization and tuning.

Given that we have a large dataset and want to balance efficiency and accuracy, a two-stage approach works best:

  • Stage 1: Candidate Generation The goal here is to narrow down the pool of potential videos from billions to thousands. At this stage, efficiency matters more than accuracy, and false positives are acceptable. We can use embedding-based models for candidate retrieval.

  • Stage 2: Scoring In this stage, the focus shifts to accurately scoring and ranking the candidate videos. Since accuracy takes priority over efficiency, we use Complex Feature Interaction Models to generate the final recommendations.

* Introduce

1. Memory-based Collaborative Filtering

核心思想: The core idea of collaborative filtering is to utilize interaction data between users and items, leveraging the similarity principle for recommendations:

  • If two users exhibit similar behaviors, they are likely to enjoy the same items (User-based collaborative filtering).

  • If two items are liked by similar users, these items are likely to be similar (Item-based collaborative filtering).

架构

Suppose we have a user-item interaction matrix, where each element represents a user's rating or implicit feedback (e.g., clicks or purchases) for an item:

  • Data Input

    • The input is a sparse user-item interaction matrix R, where Ru,i represents the interaction (e.g., rating or click) between user u and item i.

  • Model Training / Preparation

    • Build the "User-Item" Interaction List: for each user ID, retrieve recently interacted items

    • Build the "Item-Item" Similarity List:

      • Use co-occurrence counts or cosine similarity to calculate the similarity between pairs of items.

      • For each item, find its Top-K most similar items, enabling quick retrieval of the top similar items for any given item ID.

  • Inference: Prediction and Recommendation

    • Retrieve User’s Historical Behavior: Based on the user ID, use the "User-Item Interaction" list to fetch the user’s recently interacted or interested items.

    • Find Similar Items: For each item in the user’s interest list, use the "Item-Item Similarity" list to retrieve its Top-K similar items.

    • Interest Score Calculation: For all the retrieved items, calculate the user’s interest score for each item ∑jlike(user,itemj)×sim(itemj,item).

    • Ranking and Returning Results: Sort the candidate items based on their interest scores and select the Top-M items as the final recommendation results.

优缺点

  • 【Pros】Strong real-time capability: Does not require a training process; recommendations can be generated quickly based on user or item similarity.

  • 【Cons】Scalability issues: High computational cost for large datasets due to pairwise similarity calculations.

    • Use approximate nearest neighbors (ANN) techniques (e.g., k-d trees, locality-sensitive hashing) to speed up similarity computations.

  • 【Cons】Data sparsity: Performance deteriorates when the interaction matrix is sparse.

    • Combine with matrix factorization techniques to address sparsity.

2. Matrix Factorization

核心思想: Matrix factorization decomposes the user-item interaction matrix into two low-dimensional matrices (user vectors and item vectors), identifying representations of users and items in a latent space. The interest level between a user and an item is then predicted via the dot product of their corresponding vectors.

架构

Suppose we have a user-item interaction matrix, where each element represents a user's rating or implicit feedback (e.g., clicks or purchases) for an item:

  • Data Input

    • The input is a sparse user-item interaction matrix R, where Ru,i represents the interaction (e.g., rating or click) between user u and item i.

    • Since most users interact with only a small fraction of items, this matrix is highly sparse.

    • The goal is to factorize this matrix to predict the missing interactions.

  • Model Strructure: Matrix Factorization

    • We assume that users and items can be represented by low-dimensional latent vectors. The original matrix R is approximated by the product of two low-rank matrices: R≈P⋅QT where P is the user matrix, and each row Pu represents the latent vector of user u; Q is the item matrix, and each row Qi represents the latent vector of item i.

    • The predicted interaction score between user uuu and item iii is computed as the dot product of their latent vectors: R^u,i=Pu⋅QiT

  • Model Training

    • The model is trained by minimizing the error between the predicted scores and the observed scores. A common loss function is the Mean Squared Error (MSE):L=∑(u,i)∈Observed(Ru,i−Pu⋅QiT)2+λ(∥Pu∥2+∥Qi∥2)

      where λ is a regularization term to prevent overfitting.

  • Inference: Rating Prediction and Recommendation

    • After training, the model can predict the missing scores R^u,i for all user-item pairs.

    • Based on these predicted scores, the system can recommend the highest-rated items to a user or suggest items the user has not yet interacted with but is likely to prefer.

优缺点

  • 【Cons】Inability to model high-dimensional nonlinear feature interactions: Struggles to capture complex, nonlinear relationships between user preferences and item features.

    • Integrate with deep learning (e.g., NeuMF) to learn more complex feature interactions.

3. Factorization Machine (FM)

核心思想: FM learns low-dimensional embeddings for features and effectively models pairwise interactions between them, capturing latent relationships within the data.

架构

Suppose we have a feature vector that includes user, item, and other contextual information, such as user ID, item ID, user age, and purchase time:

  • Data Input

    • The input is a sparse, high-dimensional feature vector x where each element represents the value of a specific feature. For example, x may contain user ID, item ID, gender, and timestamp.

    • Features can be numerical (e.g., age) or categorical (e.g., user ID, item ID), with categorical features converted into sparse vectors using one-hot encoding.

  • Model Structure

    • Factorization Machines extend linear models by introducing interaction terms to capture high-order feature interactions. The prediction formula is as follows:

      y^=w0+∑i=1nwixi+∑i=1n∑j=i+1n⟨vi,vj⟩xixj

      where:

      • w0 is the global bias.

      • wi is the weight for the i-th feature, capturing its individual contribution.

      • ⟨vi,vj⟩=∑f=1kvi,fvj,f,f is the dot product of the latent vectors for features i and j, capturing their second-order interactions.

      • y^ is the predicted value, representing either a user's preference for an item or the probability of a certain interaction.

  • Model Training

    • The model learns parameters by minimizing the error between the predicted value y^ and the actual target value y. A common loss function is the Mean Squared Error (MSE): L=∑(x,y)(y−y^)2+λ(∥w∥2+∥v∥2) where λ is a regularization term to prevent overfitting.

  • Inference: Prediction and Applications

    • After training, Factorization Machines can be used for tasks such as:

      • Rating Prediction: Predict a user’s rating for an item (e.g., movie rating).

      • CTR Prediction: Predict whether a user will click on an ad.

      • Recommendation System: Recommend items most relevant to a user based on feature interactions.

优缺点

  • 【Pros】Robustness to sparse data: Works well with large-scale, sparse datasets and effectively models second-order feature interactions.

  • 【Cons】Limited to second-order feature interactions: Cannot capture more complex, higher-order relationships.

    • Combine with deep learning (e.g., DeepFM) to model higher-order, nonlinear feature interactions.

4. Two-Tower Model

核心思想: The core idea is to separately learn embeddings for users and items using two independent networks, and then make recommendations based on user and item embedding similarity.

架构

  • Data Input: Suppose we have user features and item features:

  • Model Structure: the model has two towers, user tower and item tower

    • User Tower

      • The user tower takes user features as input, such as age, gender, and browsing history.

      • These features are passed through embedding layers to convert categorical features into dense vectors.

      • The embedding vectors are concatenated with numerical features and fed into a deep neural network (DNN) to generate the user representation u.

    • Item Tower

      • The item tower takes item features as input, such as item ID, category, and price.

      • Similarly, the item features are passed through embedding layers, concatenated with numerical features, and fed into a DNN to generate the item representation v.

    • Similarity Calculation

      • The user vector u from the user tower and the item vector v from the item tower are matched using a similarity function, typically a dot product or cosine similarity: Score(u,v)=u⋅v

      • The calculated similarity score represents how well a user matches an item. Items with the highest scores are recommended to the user.

  • Model Training:

    • The training goal is to maximize the similarity score for positive user-item pairs (e.g., items the user interacted with) and minimize it for negative pairs (e.g., randomly sampled items the user did not interact with).

    • Common loss functions include:

      • Binary Cross-Entropy Loss: L=−∑(ylog⁡(y^)+(1−y)log⁡(1−y^)) wherey^=σ(u⋅v), and y is 1 for positive pairs and 0 for negative pairs.

      • Contrastive Loss (e.g., BPR loss): Optimizes the ranking of positive items over negative ones: L=−∑(u,i,j)log⁡σ(u⋅vi−u⋅vj) where i is a positive item and j is a negative item.

  • Inference: Prediction and Recommendation

    • For a given user, compute similarity scores for all candidate items.

    • Rank the items based on the scores and recommend the top-ranked items to the user.

优缺点

  • 【Cons】Loss of interaction information: Training user and item towers separately may result in missing important user-item interaction details.

    • Add interaction modules (e.g., dot-product or attention layers) to enhance the fusion of user and item features.

5. Item2Vec

核心思想: Item2Vec leverages sequences of user interactions (e.g., purchases, clicks) and their frequencies to capture item-to-item similarities and embeding items into a low-dimensional representations.

Example: If a user frequently purchases items A and B, Item2Vec will map A and B to nearby positions in the vector space, enabling more accurate recommendations of items similar to A.

架构

Suppose we have user-item interaction data, such as a sequence of items clicked or purchased by users:

  • Data Input

    • The input consists of user interaction sequences (e.g., a user interacted with items A, B, C, and D in order).

    • Using a sliding window, we generate item-context pairs from the sequence. For example, with a sequence [A,B,C,D] and a window size of 2, we generate pairs such as (A,B),(A,C),(B,C),(B,D),(C,D).

  • Model Training: Embedding Learning

    • Each item is represented by a unique embedding vector v.

    • Using the Skip-Gram model, the goal is to maximize the co-occurrence probability between the target item and its context items.

    • The loss function is contrastive loss based on negative sampling: L=−log⁡σ(vtarget⋅vcontext)−∑i=1Nlog⁡σ(−vtarget⋅vnegative,i)

  • Inference: Prediction and Recommendation

    • Item Similarity Calculation

      • After training, each item has a fixed embedding vector.

      • The similarity between two items can be calculated using cosine similarity:

        Similarity(i,j)=vi⋅vj∥vi∥∥vj∥

    • Recommendation Output：For a given target item, find the most similar items in the embedding space based on cosine similarity. Recommend the top-ranked items to the user.

优缺点

  • 【Cons】Lack of user personalization: Only models relationships between items and ignores user-specific preferences.

    • Incorporate user features (e.g., User2Vec) to jointly model user-item interactions and improve personalization.

6. DeepWalk

核心思想: DeepWalk generates sequences through random walks on a graph, and then it leverages the sequences to capture node-to-node similarities, and embedding nodes into a low-dimensional representations.

Example: In a social network, if users A and B are connected through multiple paths (e.g., mutual friends or similar interest groups), DeepWalk will embed A and B close to each other in the vector space, making it easier to recommend them to follow one another.

架构

Suppose we have a social network or graph data, where nodes represent users and edges represent relationships:

  • Data Input

    • DeepWalk takes a graph as input, where each node represents an entity (e.g., a user or item), and edges represent relationships (e.g., friendships or co-occurrences).

    • The first step is to perform random walks on the graph to generate node sequences. A random walk starts from a node and randomly selects a neighboring node at each step, producing sequences similar to "sentences." For example, a random walk might produce [A,B,C,D,E].

  • Model Training: Embedding Learning

    • These generated node sequences are treated like word sequences in natural language. A Skip-Gram model is used to learn node embeddings.

    • The goal is to learn embeddings that bring neighboring nodes closer in the embedding space.

    • The Skip-Gram objective function aims to maximize the conditional probability of context nodes given a target node: ∏u∈V∏v∈Context(u)P(v|u)

    • Optimized using negative sampling, the loss function is:

      L=−log⁡σ(vu⋅vv)−∑i=1Nlog⁡σ(−vu⋅vnegative,i)

  • Inference: Prediction and Recommendation

    • Node Similarity Calculation

      • After training, each node is represented by an embedding vector.

      • The similarity between two nodes can be calculated using cosine similarity: Similarity(i,j)=vi⋅vj∥vi∥∥vj∥

    • Based on the node embeddings, several tasks can be performed:

      • Node Recommendation: Recommend the most similar nodes to a given node.

      • Node Classification: Use the node embeddings as features for a classification model.

      • Community Detection: Cluster nodes based on their embeddings to discover community structures in the graph.

优缺点

  • 【Cons】Limited global information: Only captures local graph structures through random walks, missing global structural patterns.

    • Integrate with GNNs (e.g., GCN or GraphSAGE) to better model global graph structures.

  • 【Cons】Simple random walk strategy: Existing random walk strategies may not fully capture diverse relationships between nodes.

    • Use biased random walks (e.g., Node2Vec) to capture more nuanced and diverse node relationships.

7. Wide & Deep Model

核心思想: The Wide & Deep Model consists of two parts: the Wide part and the Deep part. The Wide part captures explicit feature interactions to remember past user preferences, while the Deep part captures high-order feature interactions, uncovering latent needs that users might not have explicitly expressed. By combining these two, the model offers both accuracy and novelty in recommendations.

Example: In an e-commerce scenario, the Wide part leverages a user’s purchase history (e.g., frequent purchases of electronics) to strengthen recommendations for known preferences, while the Deep part analyzes implicit features (e.g., browsing time, product descriptions) to discover and recommend new products the user might be interested in.

架构

  • Data Input: The input data includes numerical features and categorical features for users and items.

  • Model Structure: The model consists of two parts, wide part and deep part.

    • For the wide part: It takes the categorical features as input. These features are passed through a cross-product layer to generate feature crosses, which capture explicit feature interactions. The results are then passed through a linear layer to generate the wide part's output.

    • For the deep part: It takes both numerical and categorical features. Categorical features first go through embedding layers to convert them into dense vectors. These embeddings are then concatenated with the numerical features, and the combined features are fed into a neural network (DNN) to generate the deep part's output.

    • Combining the outputs: The outputs of the wide part and the deep part are stacked together, then passed through a weighted linear layer. Finally, an activation function (e.g., sigmoid) is applied to produce the final result.

  • Model Training

    • The training objective is to minimize the difference between the predicted value and the actual label.

      • If the label is a binary class, use cross-entropy loss: L=−∑(ylog⁡(y^)+(1−y)log⁡(1−y^))

      • If the label is a number, it is a regression tasks, sse Mean Squared Error (MSE) loss: L=1N∑i=1N(yi−y^i)2

  • Inference: Prediction and Recommendation

    • For each candidate item, we first use the model to compute a prediction score.

    • The model ranks items based on their predicted scores.

    • Items with the highest scores are recommended to the user.

优缺点

  • 【Cons】Dependency on feature engineering: The Wide component requires extensive manual feature design, which increases development cost.

    • Use automated feature crossing techniques (e.g., CrossNet in Deep & Cross Networks) to generate feature interactions automatically.

8. Deep Interest Network (DIN)

核心思想: DIN leverages an attention mechanism to emphasizes recent actions to better capture the user’s real-time preferences. It focuses on the user’s historical behaviors that are most relevant to the current candidate item.

Example: If a user recently browsed several smartphones, DIN will prioritize those browsing behaviors most similar to the current candidate product, enhancing recommendation precision.

架构

Suppose we have a user’s historical behavior data (e.g., items the user clicked, viewed, or purchased) and the current target item to be recommended:

  • Data Input

    • The input consists of two parts:

      • User historical behavior sequence: A sequence of items {i1,i2,…,in} that the user interacted with, along with their features (e.g., category, brand, price).

      • Target item features: The features of the current item to be recommended (e.g., item ID, category, price).

    • The goal is to predict the user’s interest y^ in the target item based on their historical behavior and the target item’s features (e.g., click probability or purchase probability).

  • Model Structure

    • DIN captures the dynamic user interests by modeling the relevance between the user’s historical behavior and the target item. The main steps include:

      1. Behavior sequence embedding: Each item in the user’s historical behavior sequence is embedded into a dense vector representation through an embedding layer.

      2. Interest extraction with attention: The historical behavior sequence is processed using an

        attention mechanism, which assigns different importance weights to each historical item based on its relevance to the target item. For example, if the target item is an "electronic device," the model will focus more on historical behaviors related to "electronics."

      3. Feature concatenation and deep network: The weighted behavior features are concatenated with the target item features and fed into a deep neural network (DNN) to capture high-order feature interactions. The output is a score representing the user’s interest in the target item.

  • Model Training

    • DIN is trained by minimizing the error between the predicted interest y^and the actual label y (e.g., whether the user clicked the item).

    • The common loss function is cross-entropy loss, defined as: L=−∑(ylog⁡y^+(1−y)log⁡(1−y^))

      where y is the ground truth label (1 for click, 0 for no click).

  • Inference: Prediction and Recommendation

    • Interest Score: The model predicts a user’s interest score y^ for the target item.

    • Personalized Recommendation: Based on the interest scores, the system recommends the highest-scoring items to the user, dynamically adapting to the user’s real-time interests.

优缺点

  • 【Cons】Limited representation of diverse interests: Struggles to model users with multiple distinct interests, especially when their preferences shift frequently.

    • Introduce multi-interest modeling (e.g., DIEN) to capture the evolution of user interests and improve diversity in interest representation.

9. Graph Neural Network (GNN)

核心思想: For each node, GNN aggregates features and relationships from its “neighbors” (other users or items directly connected to it) to learn an efficient embedding representation. Recommendation systems utilize these embeddings to predict the similarity between nodes, enabling highly accurate recommendations.

架构

Suppose we have a user-item interaction graph, where nodes represent users or items, and edges represent interactions such as clicks, ratings, or purchases:

  • Data Input

    • The input data includes:

      • Graph structure: Consisting of user nodes, item nodes, and edges representing interactions.

      • Node features: Attributes of users and items (e.g., user age, gender; item category, price).

    • The graph can be represented by an adjacency matrix A or an edge list, and each node has an initial feature vector X.

  • Model Structure

    • The core idea of GNNs is to perform “message passing” to update node features and learn richer representations of the graph.

    • Message Passing: Each node collects information from its neighboring nodes.

    • Feature Update: The collected information is combined with the node’s own features to generate a new representation. This process is repeated over multiple layers so that each node's representation incorporates not only its own information but also its neighbors’ and the graph’s global structure.

    • Simply put, it’s like each node “consulting” its neighbors and using their insights to improve its own representation.

  • Model Training

    • The goal of GNNs in recommendation is to learn high-quality embeddings for users and items, which can be used to predict interaction probabilities.

    • The loss function depends on the task. For example:

      • Binary classification (e.g., click prediction): Cross-entropy loss.

      • Rating prediction: Mean Squared Error (MSE) loss.

    • Example loss function: L=∑(u,i)∈Observed(yu,i−y^u,i)2+λ∥Θ∥2 where yu,i is the actual interaction, and y^u,i is the predicted interaction.

  • Inference：Prediction and Recommendation

    • Node Embeddings: After training, each user and item has a high-dimensional embedding vector.

    • Similarity Calculation: The preference of a user for an item can be predicted by calculating the similarity (e.g., dot product or cosine similarity) between their embeddings: y^u,i=hu⋅hi

    • Based on the predicted scores, the system recommends the most relevant items to the user.

优缺点

  • 【Cons】Over-smoothing problem: As the number of GNN layers increases, node features tend to become overly similar, leading to a loss of distinctiveness.

    • Use residual connections (e.g., Res-GNN) to mitigate over-smoothing and preserve unique node information.

10. [Diversity] Maximal Marginal Relevance (MMR)

核心思想: MMR is an iterative selection process. At each step, it tries to pick the most relevant item while avoiding redundancy, , balancing relevance and diversity in recommendations.

架构 Suppose we have a candidate list of 100 articles, already ranked by relevance. MMR further refines the selection:

  1. Initialize: Start with an empty “selected list” and put all articles in the “remaining list.” Pick the article with the highest relevance score and move it to the “selected list.”

  2. Iterative selection: Repeat the following steps until k articles are selected:

    • Score calculation: For each article in the “remaining list,” calculate a new score that combines its relevance to the user and its similarity to the articles in the “selected list.” Articles that are too similar to what’s already selected will get lower scores.

    • Select the top-scoring article: Move the article with the highest new score from the “remaining list” to the “selected list.”

11. [Diversity] Determinantal Point Process (DPP)

核心思想: DPP aims to select k items from n candidates such that the selected items have both high relevance and low similarity. DPP uses the mathematical concept of a determinant to quantify the diversity of a set.

架构 Suppose we have a candidate list of 100 articles. Each article has a relevance score and a feature vector that represents its content, such as topic or style.

  1. Initialize : Start with an empty “selected list” and put all articles in the “remaining list.” Build a similarity matrix using the feature vectors to measure pairwise similarity between articles.

  2. Iterative selection: Repeat the following steps until k articles are selected:

    • Evaluate subsets: For each article in the “remaining list,” calculate how much it improves the quality and diversity of the “selected list” if added. Diversity is measured using the determinant, where a larger value means greater diversity.

    • Select the best article: Choose the article that maximizes the combined quality and diversity of the "selected list," and move it from the "remaining list" to the "selected list."

优缺点

  • Compared to MMR, it’s more computationally complex but can optimize global diversity.

To speed up the candidate generation process, ANN can be used.

  • ANN groups embeddings of similar items into the same bucket. During a query, it compares the query embedding with the representative vector of the bucket or directly searches the bucket to quickly retrieve similar items.

Multiple Goals (Scoring)

A multi-task model works best when tasks are closely related and share similar features. A multi-expert model is better for more complex features, where tasks focus on different aspects of the data.

  • Training loss is a weighted sum of the losses from each task.

Multi-Task Model

核心思想

Muti-task model uses a shared model to capture the fundamental relationship between users and items, generating a shared feature representation. This shared representation is then used by seperate modules to predict metrics for each task .

架构

The model has two main parts:

  • Shared Layer: This is a neural network that takes all the features concatenated together as input and outputs a representation vector. This vector captures the interaction information between users and items.

  • Task-Specific Layer: Each task has its own neural network that uses the representation vector as input and outputs predictions for its specific metric.

Multi-Gate Mixture-of-Experts (MMoE)

核心思想

MMoE uses several independent "expert modules" to capture different patterns or characteristics of the data, generating multiple feature representations. In MMoE, each task employs a specific mechanism to integrate these representations and makes predictions based on the integrated features.

架构

The model consists of three components:

  • Expert Layer: Composed of multiple expert modules, each of which is a neural network (NN).

    • Input for each NN: The concatenation of user features, item features, and context features.

    • Output for each NN: A feature vector that represents the expert's specific understanding of the relationship between users and items.

  • Gating Network Layer: Composed of multiple modules, each corresponding to a specific task, with each module being a neural network (NN).

    • Input for each NN: The concatenation of user features, item features, and context features.

    • Output for each NN: A vector where each value represents the weight assigned to a corresponding expert, used to combine the feature vectors from the expert layer.

  • Task-Specific Layer: Composed of multiple prediction modules, each corresponding to a specific task, with each module being a neural network (NN).

    • Input for each NN: The task-specific feature vector obtained by weighting the expert layer's outputs.

    • Output for each NN: The predicted value for each task (e.g., click-through rate, conversion rate).

 

Part 5. Evaluation

The goal of evaluation is to ensure the model aligns with business objectives and performs effectively in both offline and real-world scenarios. For a recommendation system, this involves validating relevance, diversity, and user engagement.

Offline Evaluation

Data Split

  • K-Fold Cross Validation

    • For offline evaluation, I would use K-Fold Cross Validation to ensure a robust estimate of the model's performance.

    • Specifically, I would split the data into K equal-sized folds. Each fold would be used as the validation set once, while the remaining K−1 folds serve as the training set. This process is repeated K times, and the overall performance is averaged across all folds.

    • This approach helps reduce the impact of data splits and provides a more reliable estimate of the model's generalization. If the data is imbalanced, I would use Stratified K-Fold to maintain the same label distribution across all folds.

    • K-Fold is particularly useful when data is moderately sized, as it ensures that every data point is used for both training and validation without information leakage.

  • Hold-out Validation（留出验证）

    • For offline evaluation, I would use Hold-out Validation to quickly assess the model's performance.

    • In this approach, I would split the data into a training set and a test set, typically using an 80/20 or 70/30 split. The training set is used to train the model, while the test set provides an unbiased evaluation of its performance.

    • This method is simple and efficient, especially when dealing with large datasets where a single split is sufficient to represent the overall data distribution.

    • However, I would be cautious of the potential variance introduced by a single random split. If I notice significant variance, I could perform multiple random splits and average the results to improve robustness.

  • Bootstrap（自助法）

    • For offline evaluation, I would use the Bootstrap method to estimate the model's performance and its variability.

    • Bootstrap involves sampling the training data with replacement to generate multiple bootstrap samples, each of the same size as the original dataset. The model is then trained on each bootstrap sample and evaluated on the remaining unsampled data, known as the out-of-bag (OOB) data. By repeating this process many times, we can compute the average performance and obtain confidence intervals for the model's metrics, which is particularly useful when assessing the model’s uncertainty.

    • This approach is especially beneficial in scenarios where the dataset is small, as it allows us to fully utilize the available data without needing to reserve a large portion for testing.

    • Additionally, Bootstrap can help evaluate the model’s robustness by examining its sensitivity to variations in the training data, making it a valuable tool for understanding how much the model's performance depends on specific data points.

    • By providing confidence intervals, it also gives deeper insights into the range of potential performance outcomes, which is critical in high-stakes applications where understanding model reliability is essential.

Offline Metrics

For offline metrics, except the model loss, I would use Precision@K and Recall@K to measure relevance, NDCG for ranking quality, and mAP for a comprehensive evaluation of ranking all relevant items. For diversity, I’d calculate metrics like intra-list diversity, Gini index or catalog coverage.

  • Precision@K

    • Definition: Precision@K measures the proportion of relevant items in the top K recommendations.

    • = number of relevant items among the top k items in the output list / k

    • What it evaluates: [relevance] it focus on the accuracy of the recommendation system, ensuring that the items shown to the user are of interest.

    • Example scenario: In an e-commerce platform, users typically only look at the first 5 recommended items. Precision@5 evaluates how many of those 5 items are relevant.

  • Recall@K

    • Definition: Recall@K evaluates the proportion of the user’s relevant items that appear in the top K recommendations.

    • = number of relevant items among the top k items in the output list / total relevant items

    • What it evaluates: [relevance] It emphasizes the coverage of the recommendation system, ensuring that all user interests are considered.

    • Example scenario: In a video streaming platform, users might have diverse interests, and Recall@K checks whether the recommended list captures all major interest points.

    • Note: precision@k and recall@k need an appropriate k.

  • mAP (Mean Average Precision)

    • Definition: mAP (Mean Average Precision) is the mean of the AP (Average Precision) scores across all users or queries. AP is calculated as the average of Precision@k values at all positions where relevant items appear in the recommendation list.

    • What it evaluates: [relevance + ranking quality] It provides a comprehensive ranking assessment, considering both order and relevance.

    • 缺点：mAP is for binary relevance, NDCG is for continuous relevance. (binary：one item is considered either relevant or not; continuous: relevant degree can be 0, 1, 2 .. 5)

    • Example scenario: In a search engine, users might scroll through multiple pages of results. mAP evaluates how well the relevant results are distributed throughout the list.

  • NDCG@K (Normalized Discounted Cumulative Gain)

    • Definition: This metric measures the ranking quality of an output list and shows how good the ranking is, compared to the ideal ranking. DCG@K evaluates the quality of the ranking by assigning higher weights to the relevance scores of items that appear earlier in the recommendation list. IDCG@K is the ideal number.

    • Example: true relevance score of the ranked items [0, 5, 1, 4, 2]

      DCGk=∑i=1krelilog2⁡(i+1)=0log2⁡(2)+5log2⁡(3)+1log2⁡(4)+4log2⁡(5)+2log2⁡(6)=6.151

      IDCGk=∑i=1krelilog2⁡(i+1)=5log2⁡(2)+4log2⁡(3)+2log2⁡(4)+1log2⁡(5)+0log2⁡(6)=8.9543

      nDCGk=DCGkIDCGk=6.1518.9543=0.6869

    • 缺点：ground truth relevance scores is not always available.

    • What it evaluates: [ranking quality] It focuses on the position of relevant items, rewarding recommendations where highly relevant items are ranked higher.

    • Example scenario: In a news app, users tend to click on the first few articles. NDCG@K helps ensure that the most relevant news is ranked at the top.

  • Intra-List Diversity (ILD)

    • measures how different the items in the recommendation list are from each other. It evaluates the recommendation variety, helping to prevent repetitive or overly similar content.

  • The Gini Index

    • measures the balance in the distribution of recommendations across all items in the system. It fairness and diversity of item exposure, helping to prevent over-recommendation of popular content.

  • Coverage

    • Item Coverage measures the proportion of items recommended) and User Coverage measure the proportion of users receiving relevant recommendations).

    • It evaluates the breadth and inclusivity of the recommendation system, ensuring it leverages a wide range of items and meets the needs of a broad user base.

Part 6. Serving

After defining the appropriate metrics and training a model, the next step is deploying the model to production to serve users. Serving users effectively requires an ML system with various components workingtogether. In general, there are three main pipelines : data preparation, continual learning, and prediction.

For the data part, to serve users effectively, the system categorizes features into two types: Batch Features and Online Features.Batch features are static and rarely change, such as an item's category. These can be precomputed and stored in a Feature Store for efficient retrieval during serving.Online features, on the other hand, are dynamic and frequently updated, like time-based data. These need to be computed in real-time.

The primary responsibilities of the Data Preparation Pipeline are: (1) compute both batch and online features; (2) generate training data for model fine-tune. Therefore, the pipeline takes two inputs real-time data and historical data stored in the Data Lake. Then, both inputs are used to compute batch and online features. Once the batch features are computed, they are stored in the Feature Store for future use. Online features, combined with the batch features retrieved from the Feature Store, are then processed in the Dataset Generation module to create training data, which will be input to continual learning pipeline.

The continual learning pipeline is responsible for fine-tuning the model on new training data, evaluating its performance, and, if the new model improves the metrics, deploying it and uploading it to the Model Registry [indexing the embedding].

The prediction pipeline consists of three components: candidate generation for narrowing down the candidates from potentially billions to thousands , scoring which outputs a ranked item list, and re-ranking. This component re-ranks the videos by adding additional criteria or constraints. For example, we can increase the recommendation diversity by removing very similar item from the list.

(Data) Long-Tail Effect

  • Problem

    • Recommendation systems often suffer from long-tail effect: a small number of items account for most of the clicks, while the majority of items have very few clicks.

    • As a result, the representations of high-click items are well-learned, but those of long-tail items are poorly learned.

    • To address this, self-supervised learning is used for data augmentation to better capture the representations of long-tail items.

  • Solution: Self-supervised Learning and Contrastive Learning

    • Suppose each item has a feature vector. This approach applies two different feature transformations to each item, generating two sets of vectors. During training, it maximizes the similarity between the transformed vectors of the same item while minimizing the similarity between vectors of different items.

(Training Sample) Sample Matching Pattern

  • Pointwise Match

    • Treat recommendation as a binary classification task.

  • Hope output (e.g., cos⁡(a,b)) of positive/negative samples close to +1/-1

  • Maintain a positive-to-negative sample ratio of 1:2 or 1:3.

  • Pairwise Match

    • Each data point consists of: a user (feature vector a), the user's positiev sample ( b+); negative sample (b−)

  • Hope cos⁡(a,b+) greater than cos⁡(a,b−)，if cos⁡(a,b+) greater than cos⁡(a,b−)+m，then no loss

  • Triplet hinge loss: L(a,b+,b−)=max{0,cos⁡(a,b−)+m−cos⁡(a,b+)}

  • Listwise Match

    • Each data point consists of: a user (feature vector a), the user's positiev sample ( b+); negative samples ( b1−,...,bn−)

    • Hope cos⁡(a,b+) as large as possible， cos⁡(a,b1−),...,cos⁡(a,bn−) as small as possible

    • Positive sample y+=1，hope s+ close to 1；negative sample y1−=...=yn−=0，hope s1−...sn− close to 0

    • loss is cross-entropy loss between y and s

(Training Sample) Positive and Negative Samples

  • Positive Samples are user-item pairs where the user exposued to and clicked the item (indicating user interest).

    • Problem: A small number of popular items tend to dominate the clicks, meaning most positive samples are popular items.

    • Solution: under-sampling popular items, over-sampling not-popular items

  • Simple Negative Samples

    • Definition: datapoint sampled from the data set not selected to be candidate

      • In most cases, not-selected-as-candidate items ≈ all items, so simple negative samples ≈ all items.

    • Sampling Methods:

      • Uniform Sampling: The issue is that most items are long-tail (unpopular). If sampled uniformly, the negative samples will mostly be unpopular items, which is unfair to unpopular items.

      • Non-Uniform Sampling: To downweight popular items, the probability of being negative items is proportional to their popularity (e.g., number of clicks).

      • Batch-Based Sampling: Suppose there are n positive samples in a batch, each user is paired with n−1 items to form negative samples. This results in a total of n(n−1) negative samples in the batch, all simple negative samples.

        • Problem: The probability of an item appearing in the batch is proportional to its popularity, so popular items are more likely to become negative samples.

        • Solution: During training, adjust the interest score to cos⁡(a,bi)−log⁡pi, where pi is the item’s popularity.

  • Hard Negative Samples

    • Definition：Items eliminated during the scoring stage (粗排，relatively hard negatives). Items with low scores in the scoring stage (精排，extremely hard negatives).

Training Data: 50% simple negative samples, and 50% hard negatives samples

Note: 召回的目标是快速找到用户可能感兴趣的物品，即区分用户 不感兴趣 和 可能感兴趣 的物品，而不是区分 比较感兴趣 和 非常感兴趣 的物品。曝光没点击的物品已经非常符合用户兴趣了，可以作为排序的负样本，不能作为召回的负样本

(Training Sample) Lacking Positive Samples

  • Hard Negative Sampling: By selecting negative samples whose features are close to those of positive samples, it enhances the model's ability to distinguish between positive and negative samples.

  • Under Sampling: removing negative samples.

  • Over Sampling: duplicating positive samples.

  • Weighted Loss: Assigning higher weights to loss of positive samples ensures the model focuses more on positive samples during training.

  • Contrastive Learning: By comparing positive and negative samples, a single positive sample can be paired with multiple negatives, maximizing its use.

(Training Update) Real-Time Updates

  • Full Batch Update

    • The model is fully updated every day using all of yesterday’s data, typically trained at midnight.

    • All data is used for one epoch each day, meaning the daily data is processed only once.

    • The updated user tower neural network and item embeddings are then deployed for online retrieval.

  • Incremental Update

    • Since user interests can change rapidly, real-time online data is collected to perform online learning, incrementally updating only the User ID Embedding (other neural network parameters remain unchanged).

    • Each night, the incrementally updated model is discarded (as shown by the red X in the diagram).

Question: Can we rely solely on incremental updates without full batch updates?

User behavior varies significantly across different time periods. Full batch updates shuffle all of a day’s data randomly, eliminating temporal/period bias.

(Model Design) Similarity Measure

  • Cosine Similarity measures directional similarity of vectors, suitable for high-dimensional sparse data

  • Pearson Correlation measures linear correlation between vectors, suitable for continuous numerical data with linear relationships

  • Euclidean Distance measures absolute straight-line distance between two points, suitable for low-dimensional continuous data, especially when features are normalized

(Evaluation) Online Evaluation Strategy

除了A/B testing，还有shallow deployment，interleaving，canary release，multi-armed bandit

  • Shadow Deployment

    • runs a new model in the background, alongside the production model, but does not expose its results to users. The new model’s predictions are logged for analysis and comparison.

    • effective for testing system integration, performance, and scalability.

    • (advantage) safe: no direct impact on users, since the results are not shown.

  • Canary Release

    • gradually rolls out the new model to a small subset of users, starting with a small percentage, and slowly increasing the exposure as the model proves reliable.

    • (advantage) low-risk: if issues arise, the new model can be rolled back quickly.

  • Interleaving Experiments

    • both the old and new models make predictions for the same user, and the results are mixed within the same recommendation list. User interactions help determine which model performs better.

    • (advantage) direct comparison: Both models are tested under identical conditions.

    • (disadvantage) limited to scenarios where interleaving predictions make sense (e.g., search results, recommendations).

  • Multi-Armed Bandits

    • dynamically allocate traffic between multiple models based on their real-time performance, favoring the model that performs best while still exploring others.

    • (advantage) balances exploration (testing new models) and exploitation (using the best-performing model).

    • (advantage) quickly adapts to changing user preferences or environments.

    • (disadvantage) May not fully explore underperforming models, risking missing potential improvements (有些模型的好表现有延时，不能及时反应).
