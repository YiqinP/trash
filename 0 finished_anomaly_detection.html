Choosing the right ML category

Fusion Method

To make accurate predictions, the system should consider all modalities. Two commonly used fusing methods to combine heterogeneous data are late fusion and early fusion.

With late fusion, each modality is processed independently, and their predictions are combined to make a final prediction. [优点] This allows independent model training and improvement. [缺点] However, it requires separate training data, which can be time-consuming and expensive. In addition, it may fail when each modality is benign but harmful content arises from the combination of benign modalities.

With early fusion, the modalities are combined first, and then the model makes a prediction. [优点] Thus, it is unnecessary to collect training data separately for each modality, we only need to collect training data for the whole model. In addition, if each modality is benign, but their combination is harmful, then the model can potentially capture this. [缺点] However, learning this task is more difficult.

To capture posts that may be harmful overall, even if each modality is benign on its own, we choose early fusion. Additionally, considering the size of training data, the model has enough data to learn the task.

Classifer

Besides the fusion method, we also need to consider the type of classifier. There are three main options: One binary classifier per harmful class, Multi-label classifier, and Multi-task classifier.

With One binary classifier per harmful class, each harmful class has its own binary classifier, making it easy to explain why a post was flagged and allowing independent model improvements. However, training multiple models is expensive and time-consuming.

With multi-label classifier, asingle model predicts probabilities for multiple harmful classes, reducing training and maintenance costs. However, since all classes share the same model, input features may not be transformed optimally for each class, potentially affecting accuracy.

With Multi-task classifier, a single model learns multiple harm classes. Muti-task model uses a shared model to capture the fundamental relationship between users and items, generating a shared feature representation. This shared representation is then used by seperate modules to predict metrics for each task. This approach is cost-effective as it uses a single model, efficient by avoiding redundant computations, and data-efficient since training data from one task benefits others, especially when data is limited.

Because of these advantages, we employ a multi-task classification method.

Part 4. Model Selection

With such features, we move to models development part. Since we adopt an early fusion approach, and our detection task involves multiple data modalities (text, images, videos, and structured data), we design a two-phase model. In the first phase, we use representation learning to extract embeddings from unstructured data. In the second phase, we concatenate these embeddings with structured features and feed them into a multi-task classifier to estimate the anomaly probability.

Text (BERT) + Image (ViT)

For the multi-task classifier, neural networks are a common choice.

  • When selecting a neural network, it is essential to design its architecture and optimize key hyperparameters, such as the number of hidden layers, activation functions, and learning rate. These hyperparameters significantly impact the model's performance and are typically fine-tuned through hyperparameter tuning.

    Hyperparameter tuning is the process of finding the optimal values for hyperparameters to achieve the best model performance. A common approach is grid search, which systematically explores different combinations of hyperparameter values. This involves training a separate model for each combination, evaluating their performance, and selecting the configuration that yields the best results.

  • To construct the training data, we use two labeling methods: manual labeling and natural labeling.

    In manual labeling, human annotators carefully review and label posts, ensuring high accuracy but at the expense of time and cost. On the other hand, natural labeling leverages user reports to generate labels automatically, enabling faster data collection but introducing some label noise.

    To balance between accuracy and efficiency, we can use naturally labeled data for training, allowing us to scale the dataset quickly, while use manually labeled data for evaluation, where label quality is a top priority.

  • Since each task is a binary classification, we use cross-entropy as the standard loss function. The overall loss is then computed by aggregating the task-specific losses.

Part 5. Evaluation

Offline Metrics

  • Precision (correct rate among hit), Recall (true positive among all positive)

  • F1 score: 2×Precision×RecallPrecision+Recall

  • PR-AUC (ROC-AUC): summarize the trade-offs between precision and recall (recall and false positive)

Online Metrics

For online evaluation, we monitor several key online metrics, each providing insight into different aspects of performance.

Prevalence measures the extent to which the system fails to prevent harmful content from appearing on the platform. [公式] It is defined as the proportion of harmful posts that were not prevented relative to the total number of posts on the platform. [缺点] While this metric helps quantify the presence of harmful content, it does not reflect the actual user impact, as it does not account for how many people viewed those posts.

To address this, Harmful Impressions provides a more user-centric perspective by measuring the proportion of total impressions that came from harmful content.

Another important consideration is Valid Appeals, which measures the extent to which the system wrongly flags content as harmful. A higher value suggests that the system may be too strict, incorrectly removing posts that should have remained on the platform. [公式] This is calculated as the percentage of reversed appeals among all flagged harmful posts.

Conversely, Proactive Rate measures how effectively the system detects and removes harmful content before users report it. [公式] It is defined as the proportion of harmful posts identified by the system before user intervention. A higher proactive rate indicates that the system can autonomously detect and remove harmful posts, reducing reliance on user reports and improving platform safety.

Finally, we track User Reports per Harmful Class, which helps analyze the distribution of user-reported harmful content across different categories.

 

Part 6. Serving

After defining the appropriate metrics and training a model, the next step is deploying the model to production. In general, there are three main components : detection service, violation enforcement service, and demoting service.

Given a new post, the detection service predicts the probability of harm.

If the detection service predicts harm with high confidence, the violation enforcement service immediately takes down a post. It also notifies the user why the post was removed.

If the harmful content detection service predicts harm with low confidence, the demoting service temporarily demotes the post in order to decrease the chance of it spreading among users.

Then, the post is stored in storage for manual review by humans. The review team manually reviews the post and assigns a label from one of the predefined classes of harm. We will use these labeled posts in future training iterations to improve the model.

 

Appendix

(Data) Training Using Imbalanced Data

One-Class Learning is a common approach for handling extreme imbalance by training a model solely on normal or anomalous samples, allowing it to learn their distribution and identify deviations during inference. Examples include One-Class SVM (OC-SVM), Isolation Forest, and Autoencoder-based methods, where reconstruction error is used to detect anomalies.

Another strategy is resampling, which includes undersampling—reducing the number of normal samples to balance the dataset—and oversampling, which increases the number of anomalies using techniques like random duplication, Synthetic Minority Over-sampling Technique (SMOTE), or data augmentation methods such as GANs or Variational Autoencoders (VAE) to generate synthetic anomalous samples.

Beyond data-level modifications, optimizing the loss function is also effective. Weighted loss functions assign higher weights to anomalous samples to increase their impact on model training, such as weighted cross-entropy or focal loss, the latter reducing the influence of easily classified samples and forcing the model to focus on hard-to-classify anomalies.

Ensemble learning offers another way to address imbalance. Boosting iteratively trains multiple weak classifiers, adjusting weights to focus more on misclassified anomalies, thereby improving detection performance over iterations. Bagging, on the other hand, trains multiple classifiers on randomly sampled subsets, and when adapted for imbalanced data, it can incorporate undersampling techniques to ensure better representation of anomalies across models. Moreover, semi-supervised learning can be used when only normal data is available, training a model to recognize deviations from normal patterns, while unsupervised learning methods, such as clustering algorithms (e.g., DBSCAN, K-Means), can identify outliers that do not fit the main clusters.

  • In boosting, the classifiers are trained iteratively. To increase the prediction accuracy, the training data in each iteration is dependent on the prediction performance of the previous classifier, the algorithm is forced to choose the misclassified samples in the previous classifier more often or weight them more heavily. Thus, the boosting algorithm can be adapted to address the imbalanced data problem by adding more weights on aberrant samples in each iteration.

  • For bagging, the algorithm first samples subsets from the original data and trains classifiers independently, then makes the final result by combining the predictions with equal weight. When it is adapted to deal with the imbalance problem, instead of random sampling, several sampling strategies such as under-sampling can be used to ensure heavier weights of aberrant samples.
