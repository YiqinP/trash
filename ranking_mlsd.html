Candidate Generation

0. Two-Tower Model Overview

For candidate generation, we use a Two-Tower Model, where the query and item are encoded separately. The query tower processes text into an embedding u, while the item tower processes images into an embedding v. The similarity between them is measured by cosine similarity, and items are ranked accordingly.

  • Query Tower (Text to Embedding)

    To convert text to embedding, we can choose between statistical methods and ML-based models.

    • Statistical methods: Suitable for small datasets, interpretable, and efficient (e.g., keyword extraction, simple text matching).

    • ML-based models: Better for large-scale data and capturing complex linguistic patterns (e.g., sentiment analysis, NER, machine translation).

      • Word2Vec:

        • 核心思想：Word2Vec assumes that a word’s meaning is determined by its context. It trains a neural network to learn the relationship between words and their surrounding context, generating word embeddings.

        • Limitation：

          • Static word embeddings, unable to handle the situation where words have multiple meanings

          • Only captures local context, unable to model long-range dependencies

          • Cannot retain word order information

          • Cannot handle out-of-vocabulary (OOV) words

      • Transformer

        • 核心思想：A Transformer-Based model models language by representing words as contextual embeddings and using self-attention to capture relationships between words across an entire sequence to understand meaning and generate richer linguistic representations.

        • Limitation：

          • High computational complexity; Requires a large amount of training data; Expensive to train and deploy,

          • Difficult to interpret

      • Strategy:

        • If the task is simple: Word2Vec may be sufficient.

        • If the task requires understanding complex contexts: Transformer-based models.

        • If the goal is to efficiently search large-scale text data, a combination of Word2Vec / Bi-Encoder (e.g., CLIP) + Transformer for re-ranking

  • Item Tower (Image to Embedding)

    To convert image to embedding, we can use CNN or Vision Transformer (ViT)

    • CNN: Efficient, requires fewer parameters, and is ideal for small to mid-sized datasets and real-time applications. However, it struggles with long-range dependencies.

    • ViT: Captures global patterns well using self-attention, but requires large datasets and high computational resources. Best for cloud-based and large-scale models.

1. [Image] Convolutional Neural Networks

  • 核心思想：A CNN (Convolutional Neural Network) analyzes images by looking at small parts at a time. It first detects simple features like edges and textures, then gradually combines them to recognize more complex patterns and objects.

  • A typical CNN consists of three key components: convolutional layers, pooling layers, and fully connected layers.

    The input is an image. First, the image passes through the the convolutional layer, where small filters (kernels) slide over the input image to detect local patterns such as edges and textures. Each filter produces a feature map that highlights specific characteristics of the image.

    Next, we use pooling layers to reduce the spatial size of the feature maps while retaining the most important information. For example, max pooling selects the most significant values, making the model more efficient and robust to small shifts in the image.

    By stacking multiple convolutional and pooling layers, the network gradually learns more complex features. The early layers detect simple edges, while deeper layers recognize object parts and entire objects.

    Finally, the extracted features are flattened into a vector and passed through fully connected layers to make the final classification decision. The last layer produces the predicted label.

2. [Image] Vision Transformer

  • 核心思想：A ViT (Vision Transformer) analyzes images by splitting them into small patches and processing them like a sequence of words in a sentence. It uses self-attention to capture relationships between patches, allowing it to understand both local details and global context.

  • ViT consists of the several components.

    The input is an image. [Draw an image divided into small patches] Instead of feeding raw pixels into the model, ViT first splits an image into fixed-size patches (e.g., 16×16 pixels). Each patch is flattened into a vector and passed through a linear embedding layer to create patch embeddings.

    [Draw a positional encoding added to the embeddings] Unlike CNNs, which inherently understand spatial relationships through convolution, ViT lacks this property. To retain positional information, ViT adds a learned positional encoding to each patch embedding before feeding them into the Transformer.

    [Draw a stack of transformer blocks] The core of ViT is the Transformer encoder, which consists of multiple self-attention layers. Each layer learns relationships between different patches, allowing ViT to model global dependencies effectively. The self-attention mechanism helps ViT recognize distant patterns within an image, which CNNs struggle to do in early layers. After self-attention, a Feedforward Neural Network (FFN) refines the learned representations by applying nonlinear transformations, helping the model capture more complex feature interactions.

    After passing through the Transformer layers, the output is the embedding of the image.

1+2. [Image] Contrastive Learning

  • This model can be trained using contrastive learning. Suppose we have a positive image pair, where the images are similar, and a negative pair, where they are dissimilar. Each image is processed by the model to produce an embedding vector. Suppose we use cosine similarity: for positive pairs, the similarity should be close to 1, while for negative pairs, it should be low (closer to -1 or 0, depending on the loss function). To train the model, we typically use a contrastive loss to optimize the model.

  • Training Data Preparation:

    • Human Judgment: Manually selecting similar images, high accuracy but expensive and time-consuming.

    • User Interactions: Uses user clicks to infer similarity, making data collection automatic and scalable, but suffers from noise and sparsity.

    • Self-Supervision: we artificially create a similar image from the query image (e.g., rotation) . No manual work is required, not noisy. But constructed data differs from the real data. In practice, similar images are visually and semantically similar, but are distinct.

  • Training Loss:

    • Contrastive Loss: aims to optimize model parameters so that similar images have embeddings close to each other, while dissimilar images are farther apart in the embedding space. The computation involves three main steps:

      • Compute Similarities: Measure the similarity between the query image and other image embeddings using dot product, cosine similarity, or Euclidean distance.

      • Softmax Normalization: Apply a softmax function to convert similarity scores into a probability distribution.

      • Cross-Entropy Loss: L=−∑iyilog⁡(y^i)

    • InfoNCE Loss: aims to maximize the similarity between positive pairs while minimizing the similarity between negative pairs in the embedding space. It achieves this by using a softmax function over similarity scores, ensuring that the positive sample receives a high probability while negative samples are pushed apart. 3. [Text] Bag of Words (BoW)

  • 核心思想：BoW converts text into fixed-dimensional vectors by counting word frequencies, without considering word order or grammatical structure.

  • This method first extract unique words from all documents to form a fixed-size vocabulary. Then, for each document, it counts the occurrences of each word from the vocabulary to create a frequency vector. In this way, each document is ultimately represented as a fixed-length sparse vector, where each dimension corresponds to a word in the vocabulary, and the value represents its occurrence count in the document.

  • Limitation

    • It does not consider the order of words in a sentence. For example, "let's watch TV after work" and "let's work after watch TV" would have the same BoW representation.

    • The obtained representation does not capture the semantic and contextual meaning of the sentence. For example, two sentences with the same meaning but different words have a totally different representation.

    • The representation vector is sparse. The size of the representation vector is equal to the total number of unique tokens we have. This number is usually very large, so each sentence representation is mostly filled with zeros.

4. [Text] Term Frequency Inverse Document Frequency (TF-IDF)

  • 核心思想：Since some words appear frequently in a text but contribute little to its meaning, we can replace raw word frequencies in the BoW model with TF-IDF. This is a numerical statistic intended to reflect how important a word is to a document in a collection or corpus.

  • TF-IDF calculates Term Frequency (TF), which represents how often a word appears in a specific document, and Inverse Document Frequency (IDF), which measures how common or rare a word is across all documents. By doing so, TF-IDF reduces the influence of frequently occurring but unimportant words, improving the distinctiveness of text representation.

  • Limitation: A normalization step is needed to recompute term frequencies when a new sentence is added.

5. [Text] Word2vec

  • There are two main training approaches:

    CBOW (Continuous Bag of Words): Predicts the target word based on its surrounding context words. It is more efficient for small datasets and trains faster.

    Skip-gram: Predicts context words given a target word. It works better on large datasets and performs well for learning representations of rare words.

  • CBOW consists of three layers: the input layer, the embedding layer, and the output layer. Assuming a window size of 5, the input consists of the two words before and after the target word.

    In input later, context words are converted to one-hot vectors.

    In Embedding Layer, each one-hot vector is mapped to a word embedding, and the embeddings of the context words are then averaged to obtain a context representation.

    In Output Layer, the context representation is projected back to the vocabulary space, and softmax is applied to compute the probability distribution over the vocabulary, predicting the target word.

  • Training: In the traditional CBOW model, the goal of the model is to predict the center word based on its surrounding context words. However, since the output layer uses a Softmax function over the entire vocabulary, the computation becomes very expensive. This is especially problematic when the vocabulary size is large.

    Negative Sampling optimizes computation by reducing the number of parameters updated:

    • It samples k negative examples and attempts to minimize their probabilities.

    • It uses the Sigmoid function to compute the probability for positive and sampled negative samples

    • Then it calculate Negative Log-Likelihood (NLL) loss, avoiding the need to compute the normalization term over the entire vocabulary.

    • It updates only the weights related to the current positive and negative samples, rather than all weights in the output layer.

    • The loss function is defined as: L=−log⁡σ(vwcTvwt)−∑i=1klog⁡σ(−vwniTvwt)

      • σ(x)=11+e−x is the Sigmoid function.

      • vwt,vwc,wni is the vector representation of the center word wt, positive sample, negative samples.

6. [Text] Pretrained Transformer-Based Models

  • A Transformer consists of several key components:

    [Draw embeddings layer] The input is a sequence of words, which is first converted into word embeddings using a learned embedding layer. Each word is mapped to a dense vector representation, capturing semantic meaning.

    [Draw positional encodings added to the embeddings] Transformers do not have a built-in sense of sequence. To incorporate word order, positional encodings are added to the embeddings before they are fed into the Transformer. These encodings help the model differentiate between different positions in a sequence.

    [Draw a stack of Transformer blocks] The core of model is the Transformer block. Each Transformer block consists of multiple layers.

    • First, multi-head self-attention enables the model to focus on different relationships between words in parallel, improving its ability to model complex dependencies.

      After self-attention, a Feedforward Neural Network (FFN) applies nonlinear transformations to refine the learned representations, allowing the model to capture more abstract linguistic features.

      Finally, residual connections and layer normalization help stabilize training and improve information flow across layers, ensuring efficient learning.

    After passing through multiple Transformer blocks, the model produces contextualized word embeddings.

  • Pretrained: BERT is pre-trained using two core tasks: Masked Language Modeling (MLM) and Next Sentence Prediction (NSP).

    • In the MLM task, BERT randomly masks a portion of the input tokens and trains the model to predict the masked tokens. The loss function is a cross-entropy loss, computed only over the masked positions.

    • In the NSP task, BERT processes a pair of text segments (sentences or chunks of text). In 50% of cases, the second segment is the actual next segment in the original text (positive pair), while in the other 50%, it is a randomly sampled segment from the corpus (negative pair). The model is trained to predict whether the second segment truly follows the first one. This is a binary classification task, also optimized using cross-entropy loss.

2. ALBEF（Align Before Fuse）

  • 核心思想：ALBEF is a multimodal model that aligns and fuses text and image information, enabling better understanding and matching of visual and textual content.

  • 架构：ALBEF first uses an vision encoder (ViT) and a text encoder (BERT) to compute initial representations of images and text, bringing the embeddings of matched text-image pairs closer while pushing apart the embeddings of mismatched pairs. After this initial alignment, ALBEF employs cross-attention to further fuse text and image features, allowing deep interaction between the two modalities and capturing more fine-grained multimodal information.

Part 6. Serving

After defining the appropriate metrics and training a model, the next step is deploying the model to production to serve users. Serving users effectively requires an ML system with various components workingtogether. In general, there are three main pipelines : continual learning pipeline, indexing pipeline, and Prediction pipeline.

The continual learning pipeline fine-tunes the model using new items and user-item interactions.

The embeddings of all platform items (videos, images, and item tags) will be pre-computes and stores in an index table to accelerat the prediction pipeline.

Thus, the primary responsibilities of the index pipeline include: (1) when a new item is created, its embedding is generated and added to the index table; (2) when a newly trained model becomes available, the pipeline re-computes all embeddings using the updated model and refreshes the index table accordingly.

The prediction pipeline consists of four components: query embedding generation, candidate generation, ranking and re-ranking. The query embedding generation component computes the embedding of the input query. The candidate generation component retrieves candidates by leveraging the nearest neighbor service, identifying items whose textual or visual embeddings are most similar to the query text embedding, and merging them into a candidate list. The ranking component produces a ranked list based on relevance. Finally, the re-ranking component refines the ranked list by incorporating business-level logic and policies.
